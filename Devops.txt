Installing and Running Apache

nano start_apache.yml

---
- name: Install and Start Apache
  hosts: localhost
  connection: local
  become: yes
  become_method: sudo
  
  tasks:
    - name: Install Apache
      apt:
        name: apache2
        state: present
    - name: Start Apache
      service:
        name: apache2
        state: started
        enabled: yes

ansible-playbook start_apache.yml

To check status of Apache Service:-
sudo systemctl status apache2

Check port 80 usage:-
sudo lsof -i:80

Start Apache:-
sudo systemctl start apache2
sudo systemctl status apache2

Application Deployment

nano app_deploy.yml

---

- name: Application Deployment
  hosts: localhost
  become: yes
  tasks:
    - name: Install Apache and PHP
      apt:
        name:
          - apache2
          - php
        state: present
        update_check: yes
    - name: Deploy a PHP application
      copy:
        content: |
          <?php
          echo "<h1>Hello from Ansible and PHP</h1>";
          echo "<body>This is Welcome demo</body>";
          ?>
        dest: /var/www/html/index.php
    - name: Restart Apache
      service:
        name: apache2
        state: restarted

docker volume create myvolume1 {A Docker volume is used to store data permanently, even if the container is deleted.}

docker volume ls

docker volume inspect myvolume1 {This shows details like mount location}

docker run -v myvolume1:/data nginx {Now anything stored in /data will be saved in volume}

docker volume rm myvolume1 {It gives because the volume is used by a container.The volume must not be used by any container.}

docker run -it -v myvolume1:/data ubuntu {This command runs a container from the Ubuntu image and mounts a Docker volume inside the container.}

A. Identify Conflicting Files
Switch to the target branch and merge the other branch:
git checkout main
git merge feature-branch
Git will report a conflict. Check the status:
git status

Example output:

Unmerged paths:
  both modified: index.html

This shows that index.html has a merge conflict.

B. Edit the File to Resolve Conflicts
Open the conflicted file (index.html).

Example conflict markers:

<h1>
<<<<<<< HEAD
Welcome to My Website
=======
Welcome to Our Website
>>>>>>> feature-branch
</h1>
Decide which content to keep or combine both changes.

Resolved version:

<h1>Welcome to Our Website</h1>
Remove all conflict markers:
<<<<<<< HEAD
=======
>>>>>>> feature-branch
Save the file.
Mark the conflict as resolved:
git add index.html
C. Finalize the Merge
Commit the merge:
git commit -m "Resolved merge conflict in index.html"
Verify the repository status:
git status

Expected output:

nothing to commit, working tree clean
Push the merged changes (if required):
git push origin main
Summary of Git Commands
# Merge branches
git checkout main
git merge feature-branch

# Identify conflicts
git status

# After editing the file
git add index.html

# Finalize merge
git commit -m "Resolved merge conflict in index.html"

# Push changes
git push origin main

These steps identify the conflicting file, resolve the conflict manually in the HTML file, and complete the merge successfully.
